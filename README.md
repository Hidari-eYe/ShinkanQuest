# IPUT System Architecture: 散らばったソースファイルを復元せよ

新入生歓迎イベント用の謎解き・回遊企画特設サイトです。
サイバー・ターミナル風の世界観に合わせて、参加者が集めた7文字のソースコードを入力し、システムを復元する体験を提供します。

## ディレクトリ構成

- `app/` : ページルーティングと画面コンポーネント
  - `page.tsx`: トップページ（企画タイトルと説明）
  - `challenge/page.tsx`: 解答入力ページ（7つのソースコード入力UI）
  - `success/page.tsx`: 正解・クリアページ（順位表示とアンケート導線）
  - `failure/page.tsx`: 不正解エラーページ
  - `hint/page.tsx`: 救済用のアーカイブ・ヒントページ
  - `admin/page.tsx`: 簡易管理画面（クリア順位・時刻の閲覧）
  - `api/verify/route.ts`: 解答検証と記録保存用API
  - `api/admin/route.ts`: 管理画面アクセス用API
- `components/ui/` : 共通のアニメーション・ボタンコンポーネント群
- `lib/` : アプリケーションの共通ロジック
  - `store.ts`: `data/store.json`を用いた簡易データストア操作
- `data/` : 参加者のクリア順位データを保存するディレクトリ

## セットアップ手順

Node.js (v18以降推奨) がインストールされている環境で実行してください。

1. ターミナルでこのディレクトリ (`2026_shinkan`) に移動します。
2. 以下のコマンドを実行し、パッケージをインストールします。
   ```bash
   npm install
   ```

## 起動方法

### ローカルでの開発/テスト起動

```bash
npm run dev
```
起動後、ブラウザで `http://localhost:3000` にアクセスしてください。

### 特記事項（データ保存について）
本アプリケーションは簡易運用のために、正解時のデータを `data/store.json` に保存しています。
一度クリア状況をリセットしたい場合は、`data/store.json` の中身を空の配列 `[]` に書き直すか、ファイルを削除してシステムを再起動してください。

## 環境変数の設定・変更方法

プロジェクトルートにある `.env.local` ファイルを編集することで、正解ワードやアンケートURLを後から変更できます。
設定を変更した場合はシステム（`npm run dev`）の再起動が必要です。

**変更可能な項目:**
- `CORRECT_WORD`: 正解となる7文字のキーワード（例： `IPUT2026`）
- `ENQUETE_URL`: クリア後に誘導するアンケートのURL
- `ADMIN_PASSWORD`: 管理画面に入るためのパスワード（例： `admin_secret`）
- `HINT_1` ～ `HINT_3`: ヒントページ（アーカイブ）で表示されるヒントの文言

> **注意:**
> 英字は大文字・小文字、全角・半角を問わずに吸収して検証する仕様になっています（例：`iput2026`や`IPUT2026`、全角の`ＩＰＵＴ２０２６`でも正解となります）。

## 管理画面への入り方

1. ブラウザで `http://localhost:3000/admin` にアクセスします。
2. `.env.local` に設定したパスワード（デフォルト: `admin_secret`）を入力して「LOGIN」をクリックします。
3. 過去に正解（復元）したユーザーの順位、時刻の一覧を確認できます。

## 順位ロジックの説明

サーバー側のAPI (`/api/verify`) で `store.json` の既存レコードの一覧を読み込み、現在のクリア人数を算出して`+1`した値を今回の「順位 (Rank)」としています。
参加者が同時にアクセスし、同じタイミングで正解した場合でも、Node.jsのランタイムとファイルシステムの同期書き込みの仕様上、基本的には到達順に一意の順位が割り振られます（※ 本番環境で数千人が秒間アクセスするような要件には向いていませんが、小規模なローカル・学内イベントであれば十分動作します）。
一人が何度も正解した場合は、都度順位が加算されます。（今回は「簡易に動作させる」要件に基づき、同一IPや端末でのアクセス制限などは施していません）。

## 今後の拡張・改善余地

- **複数正解パターンの許容**: 
  `.env.local` で `CORRECT_WORD_1`, `CORRECT_WORD_2` のように複数持たせ、検証ロジックで分岐を増やすことで「隠し正解」などを作ることも可能です。
- **ランキングのリアルタイム化**:
  Pusher などを導入して、解答画面で「現在〇名がクリア！」とリアルタイムに煽る演出を追加するのも一案です。

## 本番公開（Vercel + Vercel Postgres）へのデプロイ手順

このプロジェクトはローカルのSQLiteから、クラウドの **PostgreSQL (Vercel Postgres)** に対応するように設計が変更されています。これをインターネット上に公開する手順は以下の通りです。

### 1. GitHub へのアップロード
1. ご自身の GitHub アカウントに新しい空のリポジトリを作成します。
2. ターミナルでこのフォルダを開き、以下のコマンドでソースコードをアップロードします。
   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
   git push -u origin main
   ```

### 2. Vercel への登録とデプロイ設定
1. [Vercel](https://vercel.com/) にアクセスし、GitHubアカウントでログイン（Sign In）します。
2. ダッシュボードから **「Add New...」 -> 「Project」** を選びます。
3. 先ほど作成したGitHubのリポジトリを **「Import」** します。
4. この時点ではまだデプロイせずに、**Environment Variables（環境変数）** の設定項目を開きます。
5. 手元の `.env.local` にある `CORRECT_WORD` や `ADMIN_PASSWORD`、 `HINT_1` などの値（DATABASE_URL以外）をコピーして設定します。
6. **Deploy** ボタンを押します（※この初回デプロイはデータベースがないためエラーになるか、画面は見れても送信でエラーになりますが気にしないでください）。

### 3. Vercel Postgres データベースの作成
1. Vercel のデプロイ完了画面（またはエラー画面）から、**「Continue to Dashboard」** を押してプロジェクトの設定画面に入ります。
2. 上部メニューの **「Storage」** タブをクリックし、**「Connect Store」->「Create New...」->「Postgres」** を選択します。
3. 適当なデータベース名をつけて作成し、このプロジェクトに **Connect** します。
4. Vercelが自動的に必要な環境変数（`DATABASE_URL`など）をプロジェクトにセットしてくれます。

### 4. データベースの初期化と再デプロイ
1. チーム開発画面（またはプロジェクトの Settings -> Deployments）から再度デプロイを走らせる前に、データベースのテーブルを作る必要があります。
2. ご自身のPCのターミナルで、VercelのCLIツールを使います。
   ```bash
   npm i -g vercel
   vercel link  # 作成したVercelプロジェクトとこのフォルダを紐付けます
   vercel env pull .env.development.local  # クラウドのDBパスワードを手元に引っ張ってきます
   ```
3. その直後、手元からクラウドのDBに向かって構造をプッシュします。
   ```bash
   npx prisma db push
   ```
4. Vercelのダッシュボードに戻り、プロジェクトの右上 **「Deployments」** タブから最新のコミットの横にある点々ボタンを押し、**「Redeploy」** を実行します。

これで、発行された `<プロジェクト名>.vercel.app` のURLを新入生に配る（QRコードにする）だけで、イベント本番に向けた完璧なクラウド環境が整います！
